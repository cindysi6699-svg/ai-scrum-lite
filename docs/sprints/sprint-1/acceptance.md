# Sprint 1 验收标准(Acceptance Checklist)

> Sprint 1 Goal:跑通**单 agent 闭环**——派发 → agent 执行提 PR → 人类验收/打回 → 看板状态自动更新。
> 验收方式:逐条勾选;🛡 = 守护项(针对"AI 高吞吐易出错")必须过;🚧 = 产品红线,一票否决。

---

## 通用门禁(任何 story 先过)
- [ ] 测试全绿(CI 通过)
- [ ] 改动只在该 story 范围内(无顺手乱改)
- [ ] 有执行日志 / 可追溯
- [ ] 🚧 未点「通过验收」前,push/merge 被阻止

---

## US-1 看板与 Story 基础
- [ ] 能创建 user story,并在看板列间拖拽流转
- [ ] 状态变更有留痕(谁、何时、从哪列到哪列)
- [ ] 🛡 非法流转被拒(如跳过「待验收」直达 Done,有测试覆盖)
- **DoD:** 创建/拖动生效且留痕,状态机有测试。

## US-2 把 Story 指派给 Agent
- [ ] 能像指派人一样把 story 指派给在线 agent
- [ ] 写入指派人 / agent / 时间
- [ ] 🛡 只能指派给「在线」agent,离线指派给出明确报错
- [ ] 🛡 并发/重复指派幂等(有测试)
- **DoD:** 指派可用,边界有保护。

## US-3 Agent 自动执行并提 PR
- [ ] agent 领取后自动按 story 建分支(命名规范,处理冲突)
- [ ] 提交代码 → 跑测试 → 通过后自动开 PR(关联 story)
- [ ] story 自动进入「待验收」
- [ ] 🛡 **测试不过 = 不开 PR、不进待验收**
- [ ] 🛡 认领锁:多 agent 不能抢同一 story(有测试)
- [ ] 🛡 执行超时 / agent 卡死兜底:自动回「To Do」并告警
- [ ] 🛡 完整执行日志(每步留痕)
- **DoD:** 指派后全自动跑到 PR;不达标拦在门外;异常可追溯。

## US-4 人类验收闸门 🚧 红线
- [ ] 🚧 **push/merge 硬门禁**:未「通过验收」一律阻止
- [ ] 验收队列:待验收 story + 关联 PR diff 一屏可看
- [ ] 「通过」/「打回 + 反馈」可操作
- [ ] 打回:PR 阻止合并,story 退回「Agent 执行中」并挂反馈
- [ ] 🛡 审计:每次通过/打回记录验收人、时间、反馈
- [ ] 🛡 **绕过门禁直接调 merge API 必须失败**(代码级,非仅 UI)
- **DoD:** 没人点「通过」,任何 agent 产出都 push 不出去。

## US-5 状态自动回填
- [ ] 监听 agent 事件(开始/提 PR/被打回)→ 驱动 story 流转
- [ ] 看板状态实时刷新,无需手动拖动
- [ ] 🛡 事件与真实 git/PR 状态对账,冲突以真实状态为准 + 告警
- [ ] 🛡 乱序/重复事件下状态不被写花(有测试)
- **DoD:** 看板状态随真实进度自动走,且与 git 真相不脱节。
